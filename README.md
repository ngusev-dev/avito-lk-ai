# Тестовое задание для стажёра Frontend

# Описание принятых самостоятельных решений
1. Стартовая страница согласно ТЗ является `http://localhost:5173/ads`, поэтому при открытии `http://localhost:5173/` будет ошибка 404
2. На главной странице `/ads` реализована сортировка, без учета цены, т.к данную сортировку сервер не поддерживает.
3. По макету баннер "Требуются доработки" относится только к секции "Характеристики", но но сервер возвращает поле `needsRevision` с учетом заполнения поля "Описание", поэтому во frontend части я также учитывал заполение поля "Описание", при выводе баннера на странице с объявлением `/ads/:id`

### Инструкция по запуску проекта локально, без использования Docker
## Инструкция по запуску сервера
1. Перейдите в директорию с кодом сервера — `cd server`
2. Установите зависимости, выполнив `npm i`
3. Запустите сервер - `npm start`
4. сервер запустится на `http://localhost:8080/`

В данном проекте была настроена политика CORS и изменен порт запуска сервера на `8080`

## Инструкция по запуску frontend-приложения
1. Установить зависимости `npm i`
2. Запусть проект локально командой `npm run dev`
3. Приложение запуститься на `http://localhost:5173/`

В файле `.env` при необходимости можно изменить `BASE_URL` для обращения к серверу `API` или к серверу `Ollama`.

## Запуск LLM-модели
В рамках данного задания был выбран `Ollama`
1. Установите Ollama
2. Загрузите модель командой: `ollama pull llama3`
3. Запустить `Ollama` командой: `ollama serve`

### Инструкция по запуску через Docker
1. Установите локально Ollama
2. Загрузите модель командой: `ollama pull llama3`
3. Запустить `Ollama` командой: `ollama serve`
4. Запустить команду `docker compose up` из корня данного репозитория
5. Приложение запуститьcя на `http://localhost:4173`
6. Сервер запуститься на `http://localhost:8080`

> В файле `docker-compose.yaml` при необходимости можно настроить переменные окружения для запуска frontend и server.
