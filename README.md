# Тестовое задание для стажёра Frontend

# Описание принятых самостоятельных решений
1. Стартовая старница соглансо ТЗ является `http://localhost:5173/ads`, поэтому при открытии `http://localhost:5173/` будет 404 ошибка
2. На главной странице `/ads` реализована сортировка, без учета цены, т.к данную сортировку сервер не поддерживает.
3. По макету баннер "Требуются доработки" относится только к секции "Характеристики", но при это сервер отдает поле `needsRevision` с учетом заполнения поля "Описание",
поэтому во frontend части я также учитывал заполение поля "Описание", при выводе баннера на станице с объявлением `/ads/:id`

## Инструкция по запуску севрера
1. Перейдите в директорию с кодом сервера — `cd server`
2. Установите зависимости, выполнив `npm i`
3. Запустите сервер - `npm start`

В данном проекте была настроена политика CORS

## Инструкция по запуску frontend-приложения
1. Установить зависимости `npm i`
2. Запусть проект локально командой `npm run dev`
3. Приложение запуститься на `http://localhost:5173/`

В рамках разработки придложения был изменен порт запуска `Ollama` по умолчанию - на `:5050`. 

## Запуск LLM-модели
В рамках данного задания был выбран `Ollama`
1. Установите Ollama
2. Загрузите модель командой: `ollama pull llama3`
3. Запустить `Ollama` командой: `ollama serve`
